## Core design

All tenants live in a single app (same APK). No backend required — 100% AsyncStorage.

### Storage keys
- `@ordercontrol_tenants` — registry: `{ "AB12CD": { adminEmail, restaurantName, createdAt } }`
- `@ordercontrol_session` — current session: `{ userId, tenantId }`
- `@ordercontrol_{tenantId}_users` — users for that tenant: `{ userId: StoredUser }`
- `@ordercontrol_{tenantId}_restaurant` — restaurant config
- `@ordercontrol_{tenantId}_orders` — orders array
- `@ordercontrol_{tenantId}_chats` — conversations array

### Tenant code format
6 uppercase alphanumeric chars (no 0/1/I/O to avoid confusion): e.g. `AB12CD`
Generated by `generateTenantCode()` in AuthContext.

### Auth flow
- Admin: registers restaurant → gets tenant code → logs in → goes to `/admin`
- Customer: registers with restaurant code → validated against tenants registry → goes to `/(tabs)`
- Login: searches ALL tenants for matching email+password → returns `{ success, role }`

### Provider order in _layout.tsx (CRITICAL)
AuthProvider MUST be OUTSIDE RestaurantProvider/OrderProvider/ChatProvider
because those contexts call `useAuth()` to get `tenantId`.
```
<AuthProvider>
  <RestaurantProvider>
    <OrderProvider>
      <CartProvider>
        <ChatProvider>
```

**Why:** RestaurantContext/OrderContext/ChatContext use `useEffect([tenantId])` to reload their data when the tenant changes (login/logout).

### Password change
`changePassword(oldPassword, newPassword)` in AuthContext — validates old password against stored StoredUser.password, then updates.

### User type
`User` includes `tenantId: string` field. `StoredUser extends User` adds `password: string`.
Password stored in plaintext in local AsyncStorage (no network transmission, device-local only).
